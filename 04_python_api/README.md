# 🌐 02 API

## 🎯 Objective  
Become production-ready in **API understanding, ingestion, and validation** for real-world data pipelines.

---

## 📌 Assignment  

Use any public API (recommended options):  

- JSONPlaceholder → https://jsonplaceholder.typicode.com/  
- OpenWeather API  
- GitHub API  
- Any free public REST API  

---

## 📂 Tasks  

### A) API Fundamentals  

- What is an API  
- How API works (request → response)  
- Difference between client and server  

---

### B) Types of APIs  

Explain with examples:  

- REST API  
- SOAP API  
- GraphQL API  

---

### C) HTTP Methods  

- GET (retrieve data)  
- POST (create data)  
- PUT (update data)  
- PATCH (partial update)  
- DELETE (remove data)  

---

### D) HTTP Status Codes  

Understand and document:  

- 2xx → Success (200, 201)  
- 3xx → Redirection  
- 4xx → Client Errors (400, 401, 404)  
- 5xx → Server Errors (500)  

---

### E) Response Time  

- Measure API response time  
- Understand its importance in performance  
- Identify slow API responses  

---

### F) API Ingestion  

- Fetch data from API using Python (`requests`)  
- Convert API response (JSON) into DataFrame  
- Store or print the data  

---

### G) Data Validation  

- Schema validation (check required fields)  
- Null value handling  
- Data type validation  

---

## 🚀 Outcome  

After completing this assignment, you should be able to:  

- Work with real-world APIs  
- Build API-based data ingestion pipelines  
- Validate incoming data before processing  
- Understand API behavior in production systems
